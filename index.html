Financial Sentiment Analysis

About

The goal is to analyze the impact of financial news on the market and display the results in easy to interpret visualizations. Incorporates market data from Yahoo Finance and an archive of over 4 million financial news articles covering 6,000 stocks from 2009-2020. A sentiment analysis was performed using finBERT, a pre-trained NLP model, to identify positive, neutral, and negative news headlines. The sentiment analysis was plotted alongside market movement data to create a color-coded calendar.

FinBERT Sentiment Analysis

FinBERT is an NLP model trained to perform sentiment analysis on financial text. It is based on the BERT language model and trained specifically for the finance domain. This model was used to analyze over 1 million news articles.

Yahoo Finance

Market data was obtained from Yahoo Finance. Using yFinance, data for the daily performance of the S&P 500 was pulled.

JavaScript Visualizations

Two JS libraries were used to build the visualizations. D3.js combines powerful visualization components and a data-driven approach to DOM manipulation to create custom, dynamic, and interactive data visualizations. Chart.js is used to simple, clean, and engaging HTML5 based JavaScript charts.

Project Timeline

  • Extract:

    Yahoo Finance Data and FinBERT Sentiment Analysis

    GSPC ticker data, representing the S&P 500 Index, was pulled using yFinance Python library. This ticker symbol was used as a proxy for the U.S. stock market. Data from yFinance included the date, open, high, low, close, and volume traded per day.

    A dataset of ~4 million finance news articles was available on Kaggle. Using finBERT, a pre-trained NLP model, the texts of each headline were analyzed. The finBERT analysis inicated whether news headlines were positive, negative, or neutral. FinBERT scores text on a scale of -1 (very negative) to 1 (very positive). Texts with a score of 0 are netural.

  • Transform:

    Data Clean Up & Analysis

    A PostgreSQL schema and database were set up to store the market and sentiment data. The individual data sets were cleaned up prior to loading into PostgreSQL.

    With the data loaded into PostgreSQL, necessary aggregations could be performed (e.g., averaging news sentiment scores by day), and the data could be joined for the visualizations.

  • Load:

    Putting Everything Together with SQLAlchemy and Flask

    A Python Flask application was used to serve up data and power the JavaScript visualizations. The Flask app uses SQLAlchemy to query data from the PostgreSQL database and coverts the data into JSON format. The JSON can be accessed at the '/data' route by the JavaScript code. The '/home' route accesses the main HTML page visualizations.

  • Visualization:

    Plotting with JavaScript

    The main visualization is a financial calendar that shows market movements and news sentiment using a color scale from green to pink. The two data sets are shown as connecting triangles, which together form a square. Each square represents a day. Green indicates positive market movement and positive news, and pink indicates the opposite. This visualization was created using D3.js.

    The other two visualizations were created using Chart.js. One shows daily market movement over time on a line plot. The other shows the correlation between market movement and news sentiment as a scatter plot. By default, all three visualizations show data for the 2010-2020 period. All three visualizations can also be filtered for a specific year. The filter was created using D3.js.
